To professionals, the Forexa Pro Dashboard serves as a decision-support terminal that merges traditional technical analysis with modern alternative data. Below is the detailed breakdown of the definitions and the professional workflow for using the system via GitHub Pages.

1. Core Component Definitions

RSI (Relative Strength Index) - The Momentum Oscillator

In the Forexa Pro context, the RSI is a technical momentum indicator that measures the speed and change of price movements.

  • Scale: 0 to 100.

  • Overbought (>70): Indicates the asset may be overvalued and a downward correction or "Sell" signal is approaching.

  • Oversold (<30): Indicates the asset may be undervalued and an upward "Buy" recovery is likely.

  • Professional Use: Professionals look for divergence (e.g., price makes a new high, but RSI does not), which suggests the current trend is weakening.

Sentiment Analysis - The Psychological Filter

Unlike RSI, which only looks at past price data, Sentiment Analysis uses Natural Language Processing (NLP) to scan news, social feeds, and reports.

  • Bullish Sentiment: High positive frequency in news/social data, suggesting "Greed" or "Confidence."

  • Bearish Sentiment: High negative frequency, suggesting "Fear" or "Panic."

  • Professional Use: Sentiment is used as a confirmation tool. If RSI shows "Oversold" (Buy) but Sentiment is "Extreme Fear," a professional will wait for sentiment to neutralize before entering, avoiding a "falling knife" scenario.

2. The Professional User Menu Workflow

When accessing Forexa Pro via GitHub Pages, professionals navigate the backend and frontend using a specific hierarchy to ensure data integrity.

Step A: Handshake & Tunnel Verification (Backend Page)

Before trading, a professional user must ensure the "Engine" is alive.

  1. Service Indicator: Check the top right of the backend.html page. It must show ● SERVICE: ONLINE.

  2. Tunnel Endpoint: Ensure the Ngrok or Gist URL is active. This is the "bridge" between the Python AI (running on Colab/Servers) and your browser.

  3. Live Console: Monitor the Live System Console. Professionals look for "Handshake Successful" logs to confirm that the XGBoost and LSTM models are feeding live data.

Step B: Signal Interpretation (Index/Terminal Page)

The main dashboard is where the execution happens.

  1. The Pred-Status Pill: This is the "Final Decision" output. It combines RSI, MACD, and Sentiment into a single BUY or SELL recommendation.

  2. Live Graphing: Use the Chart.js visualizer to track price action against the AI’s predicted "Target Price."

  3. Raw API Stream: For deep analysis, professionals scroll to the JSON output to see the exact confidence_score. A "BUY" signal with 0.55 confidence is risky, while 0.85 is a high-conviction trade.

3. Professional Strategy: "The Confluence Filter"

To use the website professionally, never rely on one single indicator. Follow this Rule of Three:

Step Action Requirement
1. Technical Check RSI Must be in a recovery zone (e.g., rising from 30).
2. Sentiment Check AI Sentiment Must be Positive or Neutral for a Buy.
3. Validation Check Backend Console Ensure the model latency is low (< 2 seconds).

4. Navigating the GitHub Pages Menu

  • About: Use this to review the Model Architecture. Professionals check this to understand the "Weights" (how much the AI values RSI vs. news).

  • Backend: Keep this open in a second tab. It acts as your "Engine Room" to verify that the data you are seeing on the main page isn't "stale" (old).

  • Download Client: Used for accessing the dedicated Android/Desktop app, which provides lower latency than the web-based GitHub Pages version.

Pro Tip: If the status-indicator turns Red (OFFLINE), immediately stop trading. This indicates the tunnel has collapsed or the Google Colab engine has timed out, meaning the data on your screen is no longer live market reality.
